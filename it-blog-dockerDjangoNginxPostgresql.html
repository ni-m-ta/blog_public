Docker、Django、Nginx、PostgreSQLを用いてWebサイト制作!
今回はMichael Hermanさんのアイデアを参考にします。

この記事からわかること・できるようになること

  • docker compose の書き方がわかる
  • 実際にWebサイトを制作できる

目次

  1. Dockerの準備
  2. PostgreSQLの準備
  3. Nginxの準備
  4. Static、Media filesの準備
  5. デプロイの準備

Dockerの準備

まず始めに今回の環境についてです。

  • Docker 20.10.17 (Docker Desktop Ver.)
  • Django 3.2.6
  • Gunicorn 20.1.0
  • Ngins 1.21-alpine
  • PostgreSQL 13.0-alpine

$ mkdir DirectoryName
$ cd DirectoryName
$ mkdir app
$ python3.9 -m venv env
$ source env/bin/activate
(env)$ pip install django==3.2.6
(env)$ django-admin.py startproject FunctionName
(env)$ python manage.py migrate
(env)$ python manage.py runserver

その後http://localhost:8000/に移動すると、Djangoのスクリーンを確認できます。そこで第一段階完了。

-- app
-- FunctionName
-- __init__.py
-- asgi.py
-- settings.py
-- urls.py
-- wsgi.py
-- manage.py
-- requirements.txt
-- db.sqlite3

上記のようなフォルダ構成になっているはずです。requirements.txtがない場合は追加。またDockerがない場合はインストール(今回はDocker Desktop Ver.を使用)

appディレクトリに「Dockerfile」ファイルを追加。

FROM python:3.9.6-alpine
WORKDIR /usr/src/app
ENV PYTHONDONTWRITEBYTECODE 1
ENV PYTHONUBUFFERED 1
RUN pip install --upgrade pip
COPY ./requirements.txt
RUN pip install -r requirments.txt
COPY . .

その後ルートディレクトリに「docker-compose.yml」ファイルを追加

version '3.8'
services:
web:
build: ./app
command: python manage.py runserver 0.0.0.0:8000
volumes:
- ./app/:/usr/src/app/
ports:
- 8000:8000
env_file:
- ./.env.dev

その後「settings.py」を以下のように編集

import os
SECRET_KEY = os.environ.get("SECRET_KEY")
DEBUG = int(os.environ.get("DEBUG", default=0))
ALLOWED_HOSTS = os.environ.get("DJANGO_ALLOWED_HOSTS").split(" ")

その後「env.dev」ファイルを追加。

DEBUG=1
SECRET_KEY=foo
DJANGO_ALLOWED_HOSTS=localhost 127.0.0.1 [::1]

もう一度django スクリーンを表示できるか確認。

$ docker-compose build
$ docker-compose up -d

その後http://localhost:8000/に移動すると、Djangoのスクリーンを確認で。そこで第二段階完了。

PostgreSQLの準備

ここでファイル構成を確認。

-- app
-- FunctionName
-- __init__.py
-- asgi.py
-- settings.py
-- urls.py
-- wsgi.py
-- manage.py
-- requirements.txt
-- db.sqlite3
-- env.dev
-- docker-compose.yml

docker-compose.yml ファイルを編集。

version: '3.8'
services:
web:
build: ./app
command: python manage.py runserver 0.0.0.0:8000
volumes:
- ./app/:/usr/src/app/
ports:
- 8000:8000
env_file:
- ./.env.dev
depend_on:
- db
db:
image: postgres:13.0-alpine
volumes:
- postgres_data:/var/lib/postgresql/data
environment:
- POSTGRES_USER=root
- POSTGRES_PASSWORD=root
- POSTGRES_DB=root_dev

volumes:
postgres_data:

volumesとはデータを永続化するための保管場所です。本来コンテナを削除した際データ諸々消えますが、それを防ぐための機能がボリュームです。

また.env.devファイルを編集。

DEBUG=1
SECRET_KEY=foo
DJANGO_ALLOWED_HOSTS=localhost 127.0.0.1 [::1]
SQL_ENGINE=django.db.backends.postgresql
SQL_DATABASE=db_dev
SQL_USER=root
SQL_PASSWORD=root
SQL_HOST=db
SQL_PORT=5432

またsettings.pyのDATABASESを編集。

DATABASES = {
"default": {
"ENGINE": os.environ.get("SQL_ENGINE", "django.db.backends.sqlite3"),
"NAME": os.environ.get("SQL_DATABASE", BASE_DIR / "db.sqlit3"),
"USER": os.environ.get("SQL_USER", "user"),
"PASSWORD": os.environ.get("SQL_PASSWORD", "password"),
"HOST": os.environ.get("SQL_HOST", "localhost"),
"PORT": os.environ.get("SQL_PORT", "5432"),
}
}

ここで、envの情報をもとに構築されるようになりました。

次にDockerファイルを編集

# pull official base image
FROM python:3.9.6-alpine

# set work directory
WORKDIR /usr/src/app

# set environment variables
ENV PYTHONDONTWRITEBYTECODE 1
ENV PYTHONUNBUFFERED 1

# install psycopg2 dependencies
RUN apk update \
&& apk add postgresql-dev gcc python3-dev musl-dev

# install dependencies
RUN pip install --upgrade pip
COPY ./requirements.txt .
RUN pip install -r requirements.txt

# copy project
COPY . .

psycopg2とはPythonからPostgreSQLにアクセスするためのツールです。

requirements.txtに以下のコードを追加。

Django==3.2.6
psycopg2-binary==2.9.1

ここであらためてビルドしてみましょう。

$ docker-compose down -v(もし以前のものをビルドしたままであれば...)
$ docker-compose up -d --build
$ docker-compose exec web python manage.py migrate --noinput
